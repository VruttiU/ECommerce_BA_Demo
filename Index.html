Business Problem Statement

The client (a mid-sized retail business) lacked a unified digital sales channel, causing loss of potential customers who preferred online shopping. The existing manual order process led to frequent errors, delayed fulfillment, and poor customer experience. The goal was to design and document requirements for a fully functional e-commerce platform covering product discovery, cart management, payment processing, and order tracking.

Functional Requirement Document (FRD) Summary
FRD Detail — Cart & Payment Modules
FRD-CART-01 Cart Persistence: Logged-in users retain cart items across sessions; guest users retain cart in browser session until checkout or timeout.
FRD-CART-02 Quantity Rules: Quantity can be increased or decreased from cart; system validates stock before checkout and recalculates subtotal instantly.
FRD-CART-03 Coupon Handling: Coupon validation checks expiry, minimum cart value, eligible category, and one-coupon-per-order rule.
FRD-CART-04 Abandonment UX: Cart page shows delivery estimate, transparent charges, saved items, and clear checkout CTA to reduce drop-off.
FRD-PAY-01 Payment Modes: Checkout supports UPI, debit/credit card, and wallet in Phase 1; net banking is logged as a future backlog item.
FRD-PAY-02 Payment Security: Card data is handled by payment gateway tokenization; platform does not store sensitive card data and follows PCI-DSS expectations.
FRD-PAY-03 Failure Handling: Failed or timed-out payment redirects to retry page with order summary retained; no duplicate order is created.
FRD-PAY-04 Confirmation: Successful payment displays confirmation within 5 seconds and triggers email/SMS within 1 minute.
Use Case — Checkout Payment
Primary ActorCustomer
PreconditionCustomer has at least one item in cart and a valid delivery address selected.
Main FlowReview cart → Select address → Apply coupon → Choose payment mode → Complete gateway payment → Receive confirmation.
Alternate FlowPayment fails or times out → User returns to retry page → Cart and order summary remain available → User retries or cancels.
PostconditionOrder is created only after payment success; confirmation is sent through email/SMS and order tracking is enabled.
User Stories
Agile format — As a [role] I want [goal] so that [benefit] · With Acceptance Criteria
US-001 · Sprint 1
As a customer, I want to search for products by name or category so that I can quickly find what I am looking for without browsing the entire catalogue.
Acceptance Criteria
  • Search bar is visible on every page of the platform
  • Results appear within 2 seconds of entering a keyword
  • User can filter results by category, price range, and rating
  • System shows "No results found" with suggestions if nothing matches
Catalogue Must Have 5 Points
US-002 · Sprint 1
As a customer, I want to add products to my cart and update quantities so that I can manage my purchase list before completing the order.
Acceptance Criteria
  • Cart icon on the header shows real-time item count
  • User can increment/decrement quantity directly in the cart
  • Removing an item prompts a confirmation before deletion
  • Cart persists across sessions for logged-in users
Cart Must Have 3 Points
US-003 · Sprint 1
As a customer, I want to create an account and log in securely so that I can access my saved addresses, order history, and wishlist.
Acceptance Criteria
  • Registration requires email, phone number, and password
  • OTP verification is mandatory for new registrations
  • Password must be minimum 8 characters with one special character
  • System locks account after 5 consecutive failed login attempts
User Auth Must Have 8 Points
US-004 · Sprint 2
As a customer, I want to complete payment using UPI, card, or wallet so that I can pay using my preferred method without leaving the platform.
Acceptance Criteria
  • Checkout page shows available payment options (UPI, debit/credit card, wallet)
  • Payment confirmation is displayed within 5 seconds of transaction
  • System sends order confirmation email and SMS within 1 minute
  • Failed payment redirects to retry page, not homepage
Payment Must Have 13 Points
US-005 · Sprint 2
As a customer, I want to track my order in real time so that I can know the exact status and estimated delivery time.
Acceptance Criteria
  • Order status screen shows: Placed, Packed, Shipped, Out for Delivery, Delivered
  • Each status change triggers a push notification and email
  • Estimated delivery date is shown on the order detail page
  • User can raise a delivery issue from the tracking page
Orders Should Have 8 Points
US-006 · Sprint 3
As a customer, I want to write a product review and rating after delivery so that I can share my experience and help other buyers make informed decisions.
Acceptance Criteria
  • Review option appears only after order status is "Delivered"
  • Rating is out of 5 stars; written review is optional (max 500 chars)
  • Reviews are visible on product page within 24 hours of submission
  • User can edit/delete their review within 7 days of posting
Reviews Could Have 5 Points
US-007 · Sprint 3
As a customer, I want to save products to a wishlist so that I can return to items I am interested in later.
Acceptance Criteria
  • Wishlist icon is visible on product listing and product detail pages
  • User must be logged in to save wishlist items
  • Wishlist items remain available across devices
WishlistCould Have3 Points
US-008 · Sprint 3
As a customer, I want to compare selected products so that I can make a faster purchase decision.
Acceptance Criteria
  • User can select up to 4 products from the same category for comparison
  • Comparison screen shows price, rating, stock, and key specifications
  • User can remove an item from comparison without losing other selections
CatalogueCould Have5 Points
US-009 · Sprint 3
As a customer, I want to view recently viewed products so that I can quickly revisit products I considered earlier.
Acceptance Criteria
  • Recently viewed section displays the last 10 viewed products
  • User can clear recently viewed history
  • Products that are no longer available are marked as out of stock

As a customer, I want to apply a discount coupon at checkout so that I can receive eligible offers before payment.
Acceptance Criteria

  • Only one coupon can be applied per order

Acceptance Criteria

Acceptance Criteria

SupportShould Have5 Points

As a business owner, I want to view basic sales and cart abandonment metrics so that I can track whether the platform is improving online conversion.
